github備忘録＆＠Home資料

githubは、複数の人で同じプログラムを開発するときに便利な機能をたくさん持っています。＠Homeプロジェクトでは、よくプログラムを共同で開発するので、使えるようになりましょう。

①.初期設定
1.gitのインストール
 homeディレクトリで
 $ sudo apt-get install git

2.githubアカウントの作成
 "https://github.com/”でアカウントを作る。

3.SSHkeyの設定
 SSHとは、Secure Shell（安全なシェル）の略で、暗号や認証の技術を利用して、安全にリモートコンピュータと通信するためのプロトコルのことである。（Wikipediaより）
 今回は、まずSSHkeyという鍵を作る（この鍵は公開鍵と秘密鍵で対になっている。）。次に、githubに公開鍵を渡して、秘密鍵は自分のPCに持っておく。そうすることで、アカウントを持たない人が勝手にソースを上げることを防ぐことができる。
 
○SSHkeyの作成
ターミナルのhomeディレクトリ上で、以下のコマンドを実行する。
　ssh-keygen -t rsa -C "アカウント作成時に登録したメアド"
すると、いろいろ聞かれるので全部Enterを押す。
homeの/.sshに秘密鍵（id_rsa）と公開鍵(id_rsa.pub)が作られる。

○SSHkeyの登録
・"https://github.com/”一番右上にある下矢印をクリック
・下から2番目のSettingsをクリック
・左の一覧にある"SSH and GPG keys"をクリック
・右上の"New SSH key"をクリック
・Titleは何でもOK。Keyの部分に、先ほど作成した公開鍵をコピペする。

4.Githubにソースを上げる
○リモートレポジトリの作成
・"https://github.com/"右上の＋ボタンをクリック
・New repositoryをクリック
・Repository nameに好きな名前を入力して、Create repositoryをクリック
・一番上に出てくる"https://github.com/"から始まるURLをコピー

○作成したリモートレポジトリにプッシュ
・Githubにプッシュしたいディレクトリの中で以下のコマンドを実行する。
 git init
 git add *
 git commit -m "好きなコメント"
 git remote add origin 先ほど作ったレポジトリのURL（先ほどコピーしたもの）
 git push -u origin master
 ユーザ名とパスワードを入力してプッシュ完了
 "https://github.com/"でリモートレポジトリにソースが上がっているか確認する。

※Please tell me who you are.というエラーが出てきたら、ターミナルのhomeディレクトリで以下のコマンドを実行する。
git config --global user.email 登録した時のメアド

○ソースの変更後、もう一度リモートレポジトリにプッシュ
・git add *
・git commit -m "好きなコメント"
・git push -u origin master

以上。
